Security Platform

Repositorio central de seguridad. Proporciona workflows reutilizables, reglas Semgrep, politicas de gobernanza y templates listos para aplicar en cualquier repositorio de la organizacion.

Arquitectura del sistema

El sistema sigue un patron hub-and-spoke. Este repositorio actua como hub central; los repositorios consumer llaman a los workflows reutilizables sin copiar codigo de seguridad. 

jgutierrezdtt/
├── security-platform/          ← Este repositorio (hub)
│   ├── .github/workflows/
│   │   └── reusable/           ← Llamados por todos los repos
│   │       ├── semgrep-scan.yml
│   │       ├── dependabot-check.yml
│   │       └── slsa-build.yml
│   ├── config/semgrep/         ← Reglas Semgrep centralizadas
│   ├── docs/tutorials/         ← 10 tutoriales tecnicos
│   ├── scripts/                ← Onboarding y reporte de org
│   └── templates/consumer/     ← Templates para copiar
│
└── security-exceptions/        ← Registro de excepciones (read-only)
    ├── exceptions/global/      ← Falsos positivos para toda la org
    ├── exceptions/by-repo/     ← Excepciones por repositorio
    └── schemas/                ← JSON Schema de validacion

Flujo de un Pull Request

Cuando un desarrollador abre un PR en un repositorio consumer, el workflow de seguridad ejecuta Semgrep y Dependabot en paralelo, descarga las excepciones aprobadas, genera una tabla visual en el propio PR y bloquea el merge si hay hallazgos de severidad critica o alta sin excepcion aprobada.

Onboarding de repositorios

Escalar a repositorios existentes

El script scripts/bulk-onboard.py aplica los templates de seguridad a todos los repos de la organizacion y abre un PR en cada uno, listo para que cada equipo revise y mergee a su ritmo. 

# Simulacion sin crear PRs (recomendado primero)
python3 scripts/bulk-onboard.py --org jgutierrezdtt --dry-run

# Onboarding de todos los repos
python3 scripts/bulk-onboard.py --org jgutierrezdtt

# Solo repos especificos (fichero con uno por linea)
python3 scripts/bulk-onboard.py --org jgutierrezdtt --repos-file repos.txt

# Limitar para prueba
python3 scripts/bulk-onboard.py --org jgutierrezdtt --limit 5

Deteccion de repos nuevos

El workflow detect-unconfigured-repos.yml se ejecuta cada lunes. Si detecta repos sin el workflow de seguridad, crea un GitHub Issue asignado al security-team con la lista completa. Los repos nuevos no pueden quedar sin configurar sin que el equipo lo sepa.

Quickstart para un repositorio individual

  1. Copiar los templates

    PLATFORM=jgutierrezdtt/security-platform

gh api repos/${PLATFORM}/contents/templates/consumer \
  --jq '.[] | .path' | while read file; do
  gh api repos/${PLATFORM}/contents/${file} \
    --jq '.content' | base64 -d > "${file#templates/consumer/}"
done

  2. Configurar los secretos requeridos

    gh secret set EXCEPTIONS_READER_TOKEN --repo tu-org/tu-repo
gh secret set SEMGREP_APP_TOKEN --repo tu-org/tu-repo   # opcional

  3. Habilitar Code Scanning

    gh api repos/tu-org/tu-repo/code-scanning/default-setup \
  -X PATCH --field state=configured --field query_suite=security-extended

  4. Abrir el primer PR — el workflow se activa automaticamente

    El primer PR mostrara el comentario de Semgrep y el resumen de Dependabot.

Workflow: semgrep-scan.yml

Escaneo SAST con Semgrep. Descarga excepciones aprobadas del registro central, filtra falsos positivos, genera tabla visual en el PR y aplica security gate configurable por severidad.

InputTipoDefaultDescripcion
scan-scopestringdiffEscanear solo el diff o el repo completo
fail-on-severitystringhighSeveridad minima para fallar el check
upload-sarifbooleantrueSubir resultados al Security tab de GitHub
create-issuesbooleanfalseCrear GitHub Issues por hallazgo
report-onlybooleanfalseGenerar reporte sin bloquear el PR
exceptions-repostringjgutierrezdtt/security-exceptionsRepositorio de excepciones
SecretRequeridoDescripcion
SEMGREP_APP_TOKENOpcionalToken de Semgrep Cloud Platform
EXCEPTIONS_READER_TOKENRequeridoFine-grained PAT read-only al repo de excepciones

Workflow: dependabot-check.yml

Valida el estado de las alertas de Dependabot y bloquea el pipeline si hay vulnerabilidades criticas no resueltas. Genera tabla visual con conteo por severidad y pega un comentario sticky en el PR.

InputDefaultDescripcion
fail-on-criticaltrueFallar si hay alertas CRITICAL abiertas
fail-on-hightrueFallar si hay alertas HIGH abiertas
max-critical0Numero maximo de alertas CRITICAL permitidas
report-onlyfalseSolo reportar sin bloquear

Workflow: slsa-build.yml

Genera provenance SLSA Level 3 para artefactos de release usando el generador oficial de la SLSA Framework con Sigstore/Cosign para firma sin clave privada. Solo se ejecuta en eventos de release o tag.

InputRequeridoDescripcion
artifact-nameSiNombre del artefacto a firmar
artifact-pathSiRuta al artefacto en el runner
build-commandSiComando de build a ejecutar
go-versionOpcionalVersion de Go si el proyecto usa Go
node-versionOpcionalVersion de Node.js si aplica
python-versionOpcionalVersion de Python si aplica
